## 🎓 MODEL LITERACY

**Mixture-of-Experts (MoE) vs. Dense Models**: When Qwen ships a "35B model with 3B active parameters," that's MoE at work. Instead of activating all 35B parameters for every token, a routing network selects a small subset of specialized "expert" sub-networks for each input. The result: you get the knowledge capacity of a 35B model with the inference cost of a 3B one. The tradeoff? MoE models use more memory (all 35B parameters still need to be loaded) and can be trickier to fine-tune since you're training routing behavior alongside expertise. But for inference-constrained environments — especially local deployment — MoE is why you can now run genuinely capable models on consumer hardware.
