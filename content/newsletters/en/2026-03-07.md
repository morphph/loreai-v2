## 🎓 MODEL LITERACY

**Mixture of Experts (MoE) vs. Dense Models**: When you see "Qwen3.5-397B-A17B," that second number — A17B — means only 17 billion parameters activate per forward pass, even though the full model has 397 billion. This is Mixture of Experts: the model has many specialized "expert" sub-networks but routes each input to only a few of them. The result is frontier-level capability at a fraction of the compute cost of a dense model the same size. It's why Nvidia can quantize a 397B model to run on practical hardware — you're only computing with 17B parameters at a time, not all 397B.

## 🎯 PICK OF THE DAY

**The clinejection attack is a wake-up call for every developer using AI coding tools.** A prompt injection hidden in a GitHub issue title — not the body, the *title* — compromised 4,000 developer machines by exploiting how AI assistants automatically ingest context. This isn't a theoretical attack described in a research paper. It happened, at scale, in production. The uncomfortable truth: the same "agentic" capabilities that make AI coding tools powerful — reading issues, understanding codebases, executing actions — create attack surfaces that traditional security models don't cover. As we race to give AI agents more autonomy (see: Cursor's cloud agents, OpenAI's autonomous labs), the security conversation needs to catch up fast. Audit what your AI tools can read, and assume that any text they ingest could be adversarial. [Read more →](https://grith.ai/blog/clinejection-when-your-ai-tool-installs-another)
