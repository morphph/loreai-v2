# 中国开源军团包场，本周 HuggingFace 热榜全是中文面孔

**2026-03-02 至 2026-03-06**

打开 HuggingFace Trending，前几名清一色中国模型。Qwen 一口气丢出全家桶，Kimi-K2.5 悄悄拿下 170 万下载，DeepSeek-R1 发布两个月了还牢牢占着榜首。这不是一两个模型的胜利，是整个生态的集体冲刺。

---

## 1. Qwen3.5 全家桶：阿里一次性铺满所有赛道

不是发一个模型，是发一整个**模型家族**。**Qwen3.5** 本周一口气放出：397B-A17B 大杯 MoE、35B-A3B 小杯 MoE、代码专精的 **Coder-Next**、还有一个 TTS 语音模型。从文本到多模态到代码到语音，全覆盖。(1,152 likes | 103 万下载)

397B 总参数、17B 激活参数的 MoE 架构，意味着推理成本远低于同等能力的稠密模型。35B-A3B 的小杯版本更值得关注 — 3B 激活参数，消费级显卡就能跑，但多模态能力（图文理解）直接拉满。(846 likes | 68 万下载)

**为什么重要：** 这不是在"追赶"，这是在**定义节奏**。当 OpenAI 和 Anthropic 还在闭源模型上打功能战的时候，Qwen 团队用开源全家桶的方式覆盖了几乎所有应用场景。对国内开发者来说，从 chat 到 coding 到 TTS 可以全部用一套技术栈，生态锁定效应开始成型。

**接下来看什么：** Coder-Next 的实际编码跑分还没完全出来。如果在 SWE-Bench 上接近 DeepSeek-R1 的水平，代码领域的格局会再洗一次牌。

[详情 ->](https://huggingface.co/Qwen/Qwen3.5-397B-A17B)

## 2. Kimi-K2.5：月之暗面的多模态暗马

**Moonshot AI** 的 **Kimi-K2.5** 本周在 HuggingFace 上悄悄爆发 — 170 万下载，2200 likes，是本周下载量最高的新模型之一。没有铺天盖地的营销，纯靠社区口碑起量。(2,200 likes | 171 万下载)

这是一个多模态模型，图文理解是核心卖点。月之暗面从 Kimi 长文本起家，到 K2 开始做多模态，K2.5 显然在图像理解上做了大幅提升。171 万的下载量说明开发者已经在用脚投票。

**为什么重要：** 中国 AI 的竞争格局正在从"双雄争霸"变成"三足鼎立"。DeepSeek 做推理，Qwen 做全栈，Kimi 切多模态 — 各有各的生态位。对开发者来说，选择变多了，这是好事。

**接下来看什么：** Moonshot 一直在攒 to-C 的用户量（Kimi 聊天应用），现在开源模型也有了社区基础。如果他们开始做 API 定价竞争，国内多模态 API 市场会变得很有意思。

[详情 ->](https://huggingface.co/moonshotai/Kimi-K2.5)

## 3. DeepSeek-R1：发布两个月，依然是王

**DeepSeek-R1** 本周依然是 HuggingFace 上 likes 最多的模型 — 13,096 likes，92.8 万下载。一个发布了两个月的模型还能维持这种热度，在开源社区极其罕见。(13,096 likes | 92.8 万下载)

R1 的长尾效应来自两个方面：一是推理能力确实强，在数学和代码任务上至今没被明确超越；二是围绕它的生态已经建立起来了 — 微调版本、量化版本、各种部署方案，社区投入的沉没成本让切换变得更难。

**为什么重要：** 这验证了一个规律 — 开源模型的真正壁垒不是参数量，是**生态粘性**。R1 能持续霸榜，不只是因为它好用，更因为太多人的工作流已经绑定在它上面了。这对 Qwen3.5 和 Kimi-K2.5 是个挑战：光模型好不够，你得让开发者愿意迁移。

**接下来看什么：** DeepSeek 到现在还没放 R2。他们在等什么？猜测是在做更激进的 MoE 架构或者多模态扩展。沉默越久，期待越高。

[详情 ->](https://huggingface.co/deepseek-ai/DeepSeek-R1)

## 4. MoE 架构全面接管：稠密模型正在成为历史

本周的一个隐藏主线：**混合专家模型（MoE）已经成为开源大模型的默认架构**。Qwen3.5 的两个主力版本都是 MoE，DeepSeek 的下一代大概率也是 MoE，连 Mistral 和 Meta 也在往这个方向走。

MoE 的核心优势是"大参数、小算力" — 397B 总参数的 Qwen3.5 只激活 17B，意味着你能用 70B 模型的成本跑 400B 模型的效果。这对推理成本敏感的应用场景（Agent、长对话、批处理）是质的飞跃。

**为什么重要：** 架构选择决定了整个产业链的走向。MoE 模型对显存要求更高但计算量更低，这意味着推理硬件的采购策略要调整 — 大内存比大算力更值得投资。如果你在做模型部署方案，现在就该按 MoE 来规划。

**接下来看什么：** 量化技术和 MoE 的结合。当 MoE 模型可以被高效量化到消费级硬件上跑时，"本地部署大模型"会从极客玩具变成实际可行的生产方案。

## 5. TensorFlow 重回热榜：老牌框架的韧性

一个有意思的信号 — **TensorFlow** 本周重新登上 GitHub Trending ML 榜单。19.4 万 stars，在 PyTorch 几乎一统江湖的今天，TensorFlow 还能引发关注，说明了什么？(193,955 stars | 75,215 forks)

答案是**部署生态**。PyTorch 在研究和训练端赢了，但 TensorFlow 在边缘部署（TFLite）、浏览器推理（TensorFlow.js）、生产环境 Serving 上依然有大量存量。大模型时代的推理需求爆发，反而让 TensorFlow 的部署工具链重新变得有价值。

**为什么重要：** 框架之争不是零和游戏。当行业重心从"训练"转向"部署"，评价标准也在变。对做端侧 AI、嵌入式部署的团队来说，TensorFlow 生态的成熟度仍然是 PyTorch 短期内追不上的。

**接下来看什么：** Google 最近在 TensorFlow 上的投入明显回升。如果他们把 Gemma 系列模型的推理全面优化到 TF Serving 上，可能会拉回一批生产环境用户。

[详情 ->](https://github.com/tensorflow/tensorflow)
