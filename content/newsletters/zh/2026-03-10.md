# Introducing O3 And O4 Mini

**2026-03-10**

今日 AI 圈最值得关注的动态。

今日看点: O3 和 O4 Mini 来了，推理模型进入下一代。, Codex 正式 GA，云端编码 Agent 成标配。, GPT-5.1 Codex Max — 专为大仓库优化。.

## 🧠 发布动态

### O3 和 O4 Mini 来了，推理模型进入下一代。
OpenAI 发布新一代推理模型 **O3** 和 **O4 Mini** — O3 主打更强的链式思维推理能力，O4 Mini 则面向成本敏感场景，在不需要 O3 全部能力时提供性价比之选。两个模型一起发，说明 OpenAI 认为推理能力不是一刀切的 — 不同任务需要不同级别的"思考深度"。开发者可以先看跑分和定价，选适合自己场景的。 [详情 →](https://openai.com/index/introducing-o3-and-o4-mini/)

### Codex 正式 GA，云端编码 Agent 成标配。
从研究预览到全面可用，**Codex** 的 GA 意味着 OpenAI 认为自主编码 Agent 已经足够稳定可以投产了。如果你还在观望要不要用 AI 编码 Agent，这个信号很明确：不是要不要用的问题，是用哪家的问题。找个真实项目试一把。 [详情 →](https://openai.com/index/codex-now-generally-available/)

### GPT-5.1 Codex Max — 专为大仓库优化。
**GPT-5.1 Codex Max** 是 Codex 内部的专用模型，针对长上下文编码任务做了优化 — 大型代码仓库、多文件编辑场景下表现更好。这是把"模型好"和"Agent 好"拆开来卖的策略：模型负责理解代码，Agent 负责编排任务。 [详情 →](https://openai.com/index/gpt-5-1-codex-max/)

**Anthropic 的 Cowork 能自己造 Skill 了。** 元技能上线 — Cowork 会先采访你的工作流程，然后自动生成定制化 Skill 并附带基准测试。AI 开始造自己的工具了，这个方向值得关注。(678 likes | 60 RTs) [详情 →](https://x.com/emollick/status/2029965617814487169)

**Gemini 3.1 Flash Image 先于 Flash 本体发布。** Google 反常操作 — 图像生成专用模型比基座模型先上线，说明多模态能力在 Google 内部的优先级可能已经高于纯文本。(201 likes | 3 RTs) [详情 →](https://x.com/simonw/status/2027142536872198359)

## 🎓 模型小课堂

**推理模型（Reasoning Model）**：普通大模型拿到问题就直接回答，推理模型不一样 — 它会先"想一想"，把问题拆成步骤，一步步推导出答案。你可以把它理解为"做题先打草稿"和"直接写答案"的区别。O3 和 O4 Mini 就是这类模型，适合数学、逻辑、代码等需要严密推理的场景。代价是更慢、更贵，所以才需要 Mini 版本来平衡成本。
