# Brainstorm 工作流

你正在帮 LoreAI v2 进行需求讨论。严格按顺序执行：

## 步骤 1: 了解上下文
- 读 PRD.md 中相关章节
- 检查现有代码实现状态（scripts/、src/、skills/）
- 检查 git log 最近 10 条 commit

## 步骤 2: 提问澄清（一次一个问题）
- 目的是什么？解决什么用户问题？
- 影响哪些 pipeline / 页面？
- 有什么约束？（API 限制、token 成本、cron 时间窗口）
- 成功标准是什么？

## 步骤 3: 提出 2-3 个方案
- 每个方案：简述方法 + 优劣 + 预估工作量
- 明确推荐哪个，说明为什么

## 步骤 4: 分段展示设计
- 每段 200-300 字，确认后继续
- 覆盖：架构变更、数据流、错误处理、测试策略

## 步骤 5: 写设计文档
- 保存到 docs/plans/YYYY-MM-DD-<topic>.md
- git commit

## 硬规则
- 在人类说 "approved" 或 "批准" 之前，禁止写任何实现代码
- YAGNI：砍掉所有 "以后可能需要" 的功能
- 一次只问一个问题，优先用选择题
