# Plan 工作流

前提：docs/plans/ 下必须有已批准的设计文档。

## 步骤
1. 读取设计文档，完全理解需求
2. 扫描相关代码（scripts/、src/、skills/），标注已实现 vs 未实现
3. 将工作拆成小任务，每个任务包含：
   - 要改的具体文件路径
   - 实现逻辑描述（不需要完整代码，但要足够清晰）
   - 验证步骤：跑什么命令、期望什么结果
4. 按依赖关系和优先级排序
5. 保存为 docs/plans/YYYY-MM-DD-<feature>-impl-plan.md
6. git commit

## 任务粒度标准
- 每个任务能在一次 Claude Code 交互中完成
- Pipeline 类任务：一次改一个 stage，不要跨 stage
- 前端任务：一次改一个页面或组件
- 如果说不清 "验证步骤"，说明任务还需要继续拆

## 不做的事
- 不写任何实现代码
- 不做 "假设没实现" 的判断——先搜索代码确认
