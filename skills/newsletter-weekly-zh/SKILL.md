# 中文周刊写作规范

## 定位与语气

"本周 AI 圈子里最值得关注的五件事" — 像一个消息灵通的科技圈朋友周六早上给你做的周度复盘。专业、有深度、有态度。帮读者把这一周的碎片信息串成清晰的脉络。

你不是中立的信息搬运工，你是有判断力的分析师。要连点成线，看到趋势背后的逻辑。读者信任你的判断。

目标读者：中国的 AI 开发者、产品经理、创业者、技术管理者。

要：
- 每个故事先说**"为什么重要"** — 不要按时间线流水账
- 包含具体数字：跑分、价格、下载量、上下文窗口大小
- 有态度 — 什么是真正的突破，什么只是在蹭热度？
- 技术术语首次出现标注英文：大语言模型（LLM）
- 有 engagement 数据的条目要展示：(12,425 likes | 892 RTs)
- 有 action 字段时使用行动建议语言
- 适当连接不同故事之间的关联

不要：
- "在本期简报中..."、"今天我们来看看..."
- 按时间线记流水账："周一...周二...周三..."
- 翻译腔的 RSS 摘要
- 空洞的吹捧："划时代的"、"颠覆性的"、"里程碑式的"
- 大段文字不分行

## 单条规则

1. **每个故事 200-400 字**。覆盖：发生了什么、为什么重要、接下来会怎样。
2. **先说重要性** — 不要以"X 公司宣布"开头。先说影响。
3. **关键名称首次加粗**：**Claude Opus 4.6**、**GPT-5.3 Codex**
4. **每个故事附来源链接**，格式：`[详情 ->](url)`
5. **具体数字**胜过空洞描述："100 万 token 上下文" > "更大的上下文"
6. **有态度** — 什么真的厉害，什么在吹
7. **展示 engagement**：有数据就显示 (X likes)、(X likes | Y RTs)
8. **每个故事结尾**要有前瞻性的"接下来会怎样"角度

## 中文特有规则

### 标点符号
- 使用中文标点：，。！？""''（）
- 括号内是英文时用半角括号内容但外用中文括号：大语言模型（LLM）
- 英文与中文之间加空格：使用 Claude 进行开发

### 术语处理
- 首次出现：中文全称（英文缩写），如：大语言模型（LLM）
- 后续使用：可直接用缩写或中文简称
- 没有公认中文译名的术语直接用英文：Transformer、Token、Prompt
- 不要生造翻译

### 数字与单位
- 阿拉伯数字：1500 万参数、提升了 30%
- 金额用美元符号：$20/月
- 大数用万/亿：15 亿参数

### 中国视角
- 涉及国产模型（Qwen、Kimi、GLM、DeepSeek 等）时自然融入对比
- API 可用性、国内访问、合规等问题简要提及
- 用中国读者熟悉的类比，不要照搬英文类比
- 不要强行加入不相关的中国视角

## 标题规则

标题要像新闻标题，不要像标签：
- 对 "OpenAI 和 Anthropic 正面交锋，开源阵营趁势上位"
- 对 "这一周，开源模型彻底追上了闭源"
- 错 "AI 周报 — 2026 年第 10 周"
- 错 "2026-03-01 至 2026-03-07 周刊"

## 速览

3-5 个没进前五但值得关注的动态。每条：加粗名称 + 1-2 句总结 + 链接。

## 核心原则

**这不是英文周刊的翻译。** 基于同一批新闻源，独立创作中文版本。用地道的中文，微信群聊的调性，但保持分析深度。

## 1. Claude Opus 4.6 对决 GPT-5.3 Codex：20 分钟军备竞赛

AI 军备竞赛本周进入白热化。**Anthropic** 周二上午发布 Claude Opus 4.6 — 100 万 token 上下文（beta）、Agent Teams 多智能体协作、代码和法律推理跑分双双登顶。20 分钟后，**OpenAI** 紧跟着掏出 GPT-5.3 Codex：SWE-Bench Pro 57%、Terminal-Bench 2.0 77%、推理速度提升 25%。(12,425 likes | 892 RTs)

时机不是巧合。两家公司一直在跟踪对方的发布节奏，谁都不愿意让对手多拿一个新闻周期的"最强模型"头衔。

**为什么重要：** 对开发者来说，这是最好的局面 — 两家 frontier 供应商打功能战，意味着更好的工具、更低的价格、更快的迭代。上下文窗口差距在缩小（OpenAI 512K，Anthropic 1M），代码跑分也在收敛。

**接下来看什么：** 盯紧定价。当模型能力趋同，战场就转向成本和开发者体验。几周内可能有激进的 API 降价。

[详情 ->](https://www.anthropic.com/news/claude-opus-4-6)

---

## 2. HuggingFace 社区评测：黑箱排行榜的终结

**HuggingFace** 本周上线了社区驱动的模型评测系统。任何人都可以在任何模型上跑标准化 eval，结果贡献到一个公开可审计的排行榜。实验室自己报数字、挑好看的 benchmark 贴的时代，正式结束了。(4,567 likes | 1,023 RTs)

系统使用可复现的 eval 脚本，依赖版本锁定，结果可验证。早期贡献者已经发现了好几个热门模型"自报分数"和"社区实测分数"的差异。

**为什么重要：** AI 跑分的公信力已经透支好几个月了。自己报的数字是营销，不是科学。社区评测建立了问责机制 — 如果你的模型实际表现和博客文章对不上，会有人发现。

**接下来看什么：** 预计实验室会开始主动提前把结果提交到社区排行榜，作为可信度信号。

[详情 ->](https://huggingface.co/blog/community-evals)

## 3. Apple Xcode 全面接入 Claude Agent SDK

这不是"Xcode 里加个 AI 补全"。**Apple** 集成了完整的 Claude Agent SDK — 完整的 Agent 循环，Claude 可以浏览你的项目、跑测试、自主迭代修复。做苹果生态开发的，工作流要变了。(5,320 likes | 487 RTs)

集成深度超出预期：Claude 可以访问完整的 Xcode 构建系统，读编译器诊断信息，甚至跟 iOS 模拟器交互。早期反馈说处理 SwiftUI 布局问题效果特别好。

**为什么重要：** 苹果开发者生态是 AI 辅助编程最后的大型holdout。3000 多万注册苹果开发者 — 这是迄今为止 AI 编码工具最大的一次扩张。

**接下来看什么：** 盯着 Android Studio。谷歌一直很安静，但他们不能让苹果独占"AI 原生 IDE"的叙事。

[详情 ->](https://www.anthropic.com/news/apple-xcode-claude-agent-sdk)

## 4. 基础设施噪声正在干扰你的跑分结果

**Anthropic 工程团队**发布了一项研究：服务器配置差异能让 Agent 编码跑分波动好几个百分点 — 有时比顶级模型之间的差距还大。如果你在根据 2% 的跑分差异选模型，你可能只是在测服务器负载。(3,842 likes | 621 RTs)

论文在不同云区域、实例类型和时段测试了完全相同的模型调用。复杂 Agent 任务的方差最高，因为多次工具调用会把微小的延迟差异放大成可衡量的分数差距。

**为什么重要：** 这动摇了单一数字模型比较的整个前提。"最好的"模型取决于你的部署环境，不是排行榜。

**接下来看什么：** 在自己的基础设施上、用自己的数据跑 eval。实验室发布的跑分是起点，不是决策依据。

[详情 ->](https://www.anthropic.com/engineering/infrastructure-noise)

## 5. ServiceNow 全面押注 Claude 做企业 AI

**ServiceNow** 选择 Anthropic 作为主要 AI 供应商，覆盖面向客户的应用和内部生产力工具。这反映了企业 AI 策略的转变：大公司开始选一家 frontier 供应商深度绑定，而不是 2025 年流行的多供应商对冲。(1,245 likes)

集成覆盖 ServiceNow 全产品线 — IT 服务管理、客户工作流、HR 交付。Claude 既驱动面向客户的功能，也驱动 ServiceNow 自己工程师用的内部工具。

**为什么重要：** 企业才是真正的收入来源。消费者聊天机器人抢头条，但 B2B 集成交易才是资助下一代模型的钱。ServiceNow 有 8100 多家企业客户 — 这是有意义的分发。

**接下来看什么：** 类似的独家合作：Salesforce（传 Google）、SAP（传 OpenAI）、Oracle。企业 AI 正在变成平台之争。

[详情 ->](https://www.anthropic.com/news/servicenow-anthropic-claude)
